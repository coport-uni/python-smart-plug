# smartplugcontroller — backup of the active `main/` firmware (ESP32-S3-BOX-3)

A standalone, buildable snapshot of the repository's **active root `main/`
firmware** — the ESP32S3WebMonitor: Beszel host monitor + Claude-usage tab +
RGB usage LED — captured on **2026-06-18**. It is kept here so the firmware can
be rebuilt later, independently of the root project.

> This is a backup copy. The live firmware still lives in the repo-root `main/`
> (`project(my_box3_sensor)`). The only difference here is the CMake
> `project()` name (`smartplugcontroller`); the `main/` sources are identical.

## What's included (and what isn't)

| Included | Why |
|---|---|
| `main/` | Byte-for-byte copy of the root `main/` sources |
| `CMakeLists.txt` | Standalone project entry (`project(smartplugcontroller)`) |
| `sdkconfig.defaults` | BOX-3 board config: 16 MB flash, Octal PSRAM @ 80 MHz, USB-Serial-JTAG console, LVGL float |
| `dependencies.lock` | Pins the managed-component versions (esp-box-3, LVGL, cJSON) so a later build resolves the same versions |

Not included, by design:
- `managed_components/` — git-ignored; `idf.py build` re-downloads them per `dependencies.lock`.
- `sdkconfig` — git-ignored; it holds Wi-Fi + Beszel credentials. Set them with `menuconfig` (below).
- `build/` — regenerated by the build.

## Build & flash (later)

```powershell
idf.py set-target esp32s3
idf.py menuconfig    # Beszel monitor    -> WiFi SSID/password, Beszel URL/user/password
                     # Claude usage tab  -> CSV URL (optional)
idf.py build
idf.py -p COM<N> flash monitor
```

Wi-Fi and Beszel credentials are **not** stored in this backup — enter them via
`menuconfig` (they land in the git-ignored `sdkconfig`).

## Note: two harmless build warnings

`idf.py build` prints:

```
warning: unknown kconfig symbol 'LV_MEM_CUSTOM' assigned to 'y'
warning: unknown kconfig symbol 'LV_MEMCPY_MEMSET_STD' assigned to 'y'
```

These are LVGL-8-era options left in `sdkconfig.defaults`; LVGL 9 removed them
and `esp_lvgl_port` manages the heap, so they are no-ops. They are carried over
verbatim from the root project to keep this an exact backup. To silence them,
delete those two lines from `sdkconfig.defaults`. (See `LearnedPatterns.md`
§3.11.)
